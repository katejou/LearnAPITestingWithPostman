# Project Setup and Usage

This repository is designed to be used in conjunction with the following repository:
[quality-engineering-api-testing-with-postman](https://github.com/taylonr/quality-engineering-api-testing-with-postman.git)

The **`quality-engineering-api-testing-with-postman`** repository provides two local environments that can be launched for learning exercises. The repository's license grants permission to use it for educational purposes.

---

## Setup Instructions

1. Clone the repository:

```bash
git clone https://github.com/taylonr/quality-engineering-api-testing-with-postman.git
```

2. Install dependencies:

```bash
npm install
```

3. Start the development environment:

```bash
npm run start:dev
```

* Expected output:

```
Server is running on http://localhost:3000
```

4. Start the QA environment:

```bash
npm run start:qa
```

* Expected output:

```
Server is running on http://localhost:3001
```

---

## Postman Collections

Inside this repository, you will find the folder **`PM_Collections_I_Made`**, which contains two Postman collections I created following the instructions of Nate Taylor on Pluralsight.

**Usage:**

1. Launch the servers as described above.
2. Import the collections into the Postman application.
3. Follow the notes included in the collections and run it, which should to make the code and workflows understandable, I hope.
